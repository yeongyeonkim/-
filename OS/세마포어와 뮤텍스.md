# 개념

> 공유된 자원에 여러 개의 프로세스가 동시에 접근하면서 문제가 발생한다.<br>
따라서 공유된 자원 속 하나의 데이터는 한 번에 하나의 프로세스만 접근하게 제한해주어야 한다.<p></p>

<ul>
  <li> 세마포어 : 공유된 자원의 데이터를 여러 <b>프로세스</b>가 접근하는 것을 막는 것.<br>
  (세마포어를 2로 초기화 했다면, 임계구역에 2개 프로세스가 들어갈 수 있음.)
    <ul><p></p>
      <li>P와 V연산에 의해 <b>임계 구역</b>의 접근을 제어.<br>
      세마포어 값에 따라 이미 다른 프로세스에 의해 사용 중인지, 즉시 자원 사용 가능한지 알 수 있게 된다.</li>
    </ul>
  </li>
</ul>

<ul>
  <li>뮤텍스 : 공유된 자원의 데이터를 여러 <b>쓰레드</b>가 접근하는 것을 막는 것.
    <ul><p></p>
      <li>임계구역을 가진 쓰레드들의 Running Time이 겹치지 않게 각각 단독으로 실행되게 하는 기술<br>
      다중 프로세스들의 공유 리소스에 대한 접근을 조율하기 위해 locking과 unlocking을 사용한다.
      </li>
    </ul>
  </li>
</ul>

***

# 차이점

1. 세마포어는 뮤텍스가 될 수 있지만, 뮤텍스는 세마포어가 될 수 없다.(Mutex는 0, 1 두 개뿐인 이진 세마포어)
2. 뮤텍스의 경우 뮤텍스를 가진 쓰레드가 뮤텍스 해제 가능, 세마포어는 소유하지 않은 쓰레드도 세마포어 해제 가능.
3. <b>뮤텍스는 동기화 대상이 오직 하나 뿐일 때, 세마포어는 동기화 대상이 하나 이상일 때 사용.</b>
4. 세마포어는 시스템 범위에 걸쳐 있고 파일 시스템상 파일 형태로 존재.<br>
반면 뮤텍스는 프로세스 범위를 가지며 프로세스가 종료될 때 자동으로 clean up 된다.
***

<h2> ※ 임계구역(Critical Section)이란? </h2>
*    다중 프로그래밍 운영체제에서 여러 프로세스가 데이터를 공유하면서<br>
    수행될 때 각 프로세스에서 공유 데이터를 액세스하는 프로그램 코드 부분.<br>
    공유 데이터를 여러 프로세스가 동시에 액세스하면 시간적 차이로 잘못된 결과를 만들 수 있기 때문에<br>
    한 프로세스가 공유 데이터를 액세스하고 있을 때는 다른 프로세스는 그 데이터를 액세스 하지 못하게 해야 한다.
